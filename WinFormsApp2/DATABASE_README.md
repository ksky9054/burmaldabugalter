# Система аутентификации с зашифрованными паролями

## Описание

Приложение теперь использует **SQLite базу данных** с **BCrypt хешированием паролей** для безопасного хранения учётных данных пользователей.

## ⚠️ ОГРАНИЧЕНИЕ: Максимум 2 аккаунта

**Важно!** Система ограничена максимум 2 активными аккаунтами. После создания 2 аккаунтов регистрация новых будет невозможна.

### Поведение при лимите:
- ✏️ Форма регистрации закроется при загрузке, если лимит достигнут
- 🔘 Кнопка "Создать новый аккаунт" будет отключена на форме входа
- ⚠️ Будет показано сообщение об ошибке при попытке создания 3-го аккаунта

## Архитектура

### DatabaseManager.cs
Класс для управления базой данных SQLite, включает:
- Инициализация базы данных
- Создание новых пользователей
- Аутентификацию пользователей
- Получение информации пользователя
- Валидацию входных данных

### CurrentUser.cs
Статический класс для управления текущей сессией пользователя

### LoginForm.cs
Форма входа с интеграцией DatabaseManager

### RegisterForm.cs
Форма регистрации с проверкой данных

## Безопасность

### Хеширование паролей
- Используется **BCrypt** вместо простого хранения
- Каждый пароль хешируется уникально
- Невозможно восстановить исходный пароль из хеша

### Валидация данных
- Логин: 3-50 символов, буквы, цифры, подчёркивание
- Пароль: 4-100 символов
- Email: стандартный формат электронной почты

### База данных
- Файл `users.db` хранится локально
- SQLite обеспечивает локальное хранилище
- Таблица `Users` с полями:
  - Id (первичный ключ)
  - Username (уникальный)
  - PasswordHash (BCrypt хеш)
  - Email
  - CreatedAt (дата создания)
  - IsActive (флаг активности)

## Использование

### Регистрация нового аккаунта
1. Нажмите "Создать новый аккаунт"
2. Введите:
   - Логин (3+ символов)
   - Пароль (4+ символов)
   - Подтверждение пароля
   - Email
3. Нажмите "Создать аккаунт"
4. Вернитесь на форму входа

### Вход
1. Введите логин и пароль
2. Нажмите "Вход" или Enter
3. При успехе откроется основная форма

## Перечень пакетов

- **Microsoft.Data.Sqlite** (8.0.0) - для работы с SQLite
- **BCrypt.Net-Next** (4.0.3) - для хеширования паролей

## Примеры

### Создание пользователя
```csharp
var dbManager = new DatabaseManager();
bool success = dbManager.CreateUser("john_doe", "SecurePassword123", "john@example.com");
```

### Аутентификация
```csharp
bool isValid = dbManager.AuthenticateUser("john_doe", "SecurePassword123");
```

### Получение информации пользователя
```csharp
var userInfo = dbManager.GetUserInfo("john_doe");
if (userInfo.HasValue)
{
	Console.WriteLine($"Email: {userInfo.Value.email}");
	Console.WriteLine($"Создан: {userInfo.Value.createdAt}");
}
```

## Вопросы и ответы

**Q: Где хранится база данных?**
A: В файле `users.db` в папке с приложением (обычно `bin\Debug\` или `bin\Release\`)

**Q: Как изменить хеш-функцию?**
A: Отредактируйте методы `CreateUser` и `AuthenticateUser` в `DatabaseManager.cs`

**Q: Можно ли экспортировать данные?**
A: Да, SQLite базу можно открыть в любом SQL клиенте (DB Browser for SQLite)

**Q: Как удалить пользователя?**
A: Добавьте метод `DeleteUser` в `DatabaseManager.cs` с SQL `DELETE` запросом

## Управление ограничением на аккаунты

### Ограничение
- **Максимум 2 активных аккаунта** в системе
- Константа `MAX_ACCOUNTS = 2` в DatabaseManager.cs

### Изменение лимита
Для изменения лимита отредактируйте в `DatabaseManager.cs`:
```csharp
private const int MAX_ACCOUNTS = 2; // Измените число
```

### Методы управления лимитом
- `GetUserCount()` - количество активных пользователей
- `IsAccountLimitReached()` - достигнут ли лимит
- `GetRemainingSlots()` - сколько слотов осталось
- `GetMaxAccounts()` - максимум допустимых аккаунтов

### Сценарии при достижении лимита

**Попытка регистрации новогоаккаунта при лимите (2/2):**
1. Форма регистрации закроется при загрузке
2. Будет показано сообщение: "Достигнут лимит создания аккаунтов!"

**На форме входа:**
- Кнопка "Создать новый аккаунт" будет отключена (grey)
- Текст кнопки изменится на "Регистрация (лимит достигнут)"
- При наличии свободных слотов покажет: "Создать новый аккаунт (1/2)"

## Системная информация (SystemInfo.cs)

Класс для отображения информации о состоянии системы:
```csharp
var systemInfo = new SystemInfo();
systemInfo.DisplaySystemInfo(); // Показывает диалог с информацией
```

Выводит:
- Количество активных аккаунтов
- Максимум аккаунтов
- Оставшиеся слоты
- Статус лимита
- Визуальную прогресс-полосу
